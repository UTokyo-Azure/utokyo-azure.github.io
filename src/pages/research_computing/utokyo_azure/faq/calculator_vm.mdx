## [料金計算ツール](https://azure.microsoft.com/ja-jp/pricing/calculator/)

Microsoft Azure では料金を事前に見積もるためのツールが用意されています．一定額までは寄付金で賄われるとは言え，例えばAIサービスの利用で高機能な仮想マシンを利用するとあっという間に上限に達し不足分を予算から支出することになってしまいますので，自分がデプロイしたサービスがどのくらいの利用料で使えるのか事前に計算し，想定を超えるようであればこまめにサービスを停止させて課金を止めるなどの対応をしてください．

## 計算例

- この例は以下の構成の計算例です
  - **Virtual Machine**: 仮想マシン（OS用ストレージ含む）
  - **Azure Blob Stoarge**: データ用ストレージ

### 1. 仮想マシン

- 製品から `コンピューティング` > `Virtual Machine` を選択．
- 画面下の**見積もり**に Virtual Machine が追加される．

#### インスタンス

| **項目** | **選択** |
| -------- | ------- |
| リージョン | Japan East |
| オペレーティングシステム | Linux |
| タイプ | ubuntu |
| レベル | Standard |
| カテゴリ | High Performance Compute |
| インスタンスシリーズ | H-series |
| インスタンス | 16GPU,112G RAM,2T 一時ストレージ |

以上一式を一カ月分(730時間)
- UTokyo Azure では課金は**従量課金**のみです．節約プラン，予約プランは選択できません．
- 常時立ち上げ状態を想定しています．定期的にシャットダウン及び起動を繰り返して節約する場合は，起動している予定の時間だけ入力してください．

#### マネージドストレージ
OS用に30Gのマネージドストレージを追加

| **項目** | **選択** |
| -------- | ------- |
| レベル | Standard HDD |
| サイズ | S4(32GB) |

以上一式を１か月分

- **ストレージトランザクション**，**帯域幅**
  - 通常は見積もらなくて問題ありませんが，リージョン間に跨ぐ通信が発生する場合は通信料が発生しますので，その場合は帯域幅に条件を入力してください．できる限り同一リージョン内で構築ください．
- **サポート**：Basic のままで構いません．

#### 小計

約 1,675ドル ([参考](https://azure.com/e/2cc3b709e1114ebdb00288b099b8db5c))

### 2. データ用ストレージ

- 製品から `ストレージ` > `ストレージアカウント` を選択．
  - Azure Files の場合は，`Azure Files` を選択してください．
- 画面下の**見積もり**に Virtual Machine が追加される．

#### ストレージ仕様

| **項目** | **選択** |
| -------- | ------- |
| リージョン | Japan East |
| タイプ | ブロック Blob Storage |
| パフォーマンス | 標準 |
| ストレージアカウントの種類 | Blob ストレージ |
| ファイル構造 | フラットな名前空間 |
| アクセス層 | ホット |
| 冗長性 | LRS |

- **容量**：100TB
- 課金は UTokyo Azure では**従量課金のみ**です．予約プランは選択できません．
- **操作とデータ転送**：初期値のままで計算しています．ご自身の利用予定に合わせて更新してください．

#### 小計

約 2,000ドル ([参考](https://azure.com/e/2cc3b709e1114ebdb00288b099b8db5c))

## 計算結果

仮想マシン: **1,675 ドル** + データ用ストレージ: **2,000 ドル** = **3,675 ドル** / 月額

- 実際は利用状況や細かい条件で増減するので，おおよその目安とお考え下さい．仮想マシンは使っていない時間をシャットダウンさせておくなどで利用料を減らすことができます．（ストレージは容量で固定です）
